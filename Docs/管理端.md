# 管理端需求文档

> 基于 Filament ^5.6，地址 `/admin`，复用 `users` 表认证。
>
> 管理员账号：`admin@example.com` / `password`（已由 `AdminSeeder` 创建，role=admin）。

---

## 1. 整体架构

```
会员端（H5 / UniApp）
  └── 普通用户：注册、登录、浏览文章
  └── 作者：写文章、提交审核、查看审核状态
  └── 登录入口：API（JWT）

管理端（Filament）
  └── 管理员：admin@example.com 登录 /admin
  └── 所有管理工作在此完成
```

---

## 2. 需要创建的 Filament Resource 清单

| Resource | 优先级 | 说明 |
|---|---|---|
| ArticleResource | 高 | 文章管理 + 审核工作台 |
| SensitiveWordResource | 高 | 敏感词管理 + 导入导出 |
| CategoryResource | 中 | 分类管理（树形结构） |
| TagResource | 中 | 标签管理 |
| ReviewLogResource | 低 | 审核日志查阅（只读） |
| UserResource | 低 | 用户管理（只读） |

## 3. ArticleResource（文章管理与审核工作台）

### 3.1 列表页

**入口**：`/admin/articles`

**筛选器**：
- 按状态（全部 / draft / pending / first_pass / published / first_reject / modify_required / appealing / second_pass / second_reject）
> 原本：pending_approved / rejected / revision_required / second_review / final_rejected / unlisted 等旧命名
> 改后：与接口文档统一为 first_pass / first_reject / second_pass / second_reject 等
- 按分类
- 按提交时间范围
- 关键词搜索（标题）

**表格列**：

| 列名 | 说明 |
|---|---|
| ID | ULID |
| 标题 | 可点击跳转编辑页 |
| 作者 | 关联 user.name |
| 分类 | 关联 category.name |
| 状态 | Badge 标签，颜色区分 |
| 审核轮次 | review_count |
| 最后提交时间 | last_review_at |
| 发布时间 | published_at |
| 创建时间 | created_at |

**表格 Actions**：
- 查看文章详情（只读弹窗或侧边页）
- 快捷操作按钮（first_pass / first_reject / modify_required），仅对 `pending` 状态文章显示

### 3.2 编辑页（侧边滑页）

**显示字段**（只读）：
- 标题、封面图、分类、标签
- 正文内容（Markdown，支持预览切换）
- 作者信息
- 是否置顶、是否精选、付费价格
- 发布时间、浏览量

**可编辑字段**：
- 状态（管理员可手动修改状态，用于 publish / unlist / reject 等操作）
- 管理员备注（rejection reason）

**Actions（操作按钮）**：

| 操作 | 适用状态 | 说明 |
|---|---|---|
| 通过（first_pass） | pending | 初审通过，status → published，published_at 写入当前时间 |
| 驳回（first_reject） | pending | 初审驳回，status → first_reject，必须填写 reason |
| 要求修改（modify_required） | pending | status → modify_required，必须填写 reason，同时创建 article_snapshot 快照 |
| 二审通过（second_pass） | appealing | status → published |
| 二审驳回（second_reject） | appealing | status → second_reject，必须填写 reason |
| 下架（Unlist） | published | status → unlisted |
| 恢复上架（Republish） | unlisted | status → pending |
| 永久删除 | draft | forceDelete() |

**审核记录区域**（同一页面底部）：
- 展示该文章的所有 `article_review_logs`，包含 reviewer、action、reason、review_round、sensitive_word_hit、时间

### 3.3 状态 Badge 颜色规范

| 状态 | 颜色 |
|---|---|
| draft | 灰色 |
| pending | 蓝色 |
| first_pass | 青色 |
| published | 绿色 |
| first_reject | 红色 |
| modify_required | 橙色 |
| appealing | 紫色 |
| second_pass | 绿色 |
| second_reject | 深红 |
| unlisted | 暗灰 |
> 原本：pending_approved / rejected / revision_required / second_review / final_rejected 等旧命名
> 改后：与接口文档统一

## 4. SensitiveWordResource（敏感词管理）

### 4.1 列表页

**入口**：`/admin/sensitive-words`

**筛选器**：
- 按级别（light / moderate / severe）
- 按分组名（group_name）

**表格列**：

| 列名 | 说明 |
|---|---|
| ID | |
| 敏感词 | word |
| 级别 | level（Badge：light=黄，moderate=橙，severe=红）|
| 分组 | group_name |
| 创建时间 | created_at |

**Actions**：
- 编辑（修改 word / level / group_name）
- 删除

### 4.2 导入功能

**触发方式**：列表页顶部按钮「批量导入」

**文件格式**：
- 纯文本文件（.txt）
- 以空格分隔单元格（一个空格 = 一个词）
- 换行符视为分隔符，同一行空格之间为独立词

**示例文件内容**（实际每行按空格切分）：

```
违禁词A 违禁词B 违禁词C
违禁词D 违禁词E 违禁词F
```

上例共导入 6 个词。

**导入行为**：
- 读取整个文件内容，按空白符（空格、制表符、换行）拆分 token
- 每个 token 为一个独立敏感词
- 已存在的词（按 word 唯一约束）自动跳过（不报错，不重复插入）
- 缺失字段（level、group_name）使用导入弹窗里选择的默认值
- 导入完成后显示：共解析 N 个词，成功导入 M 个，跳过 X 个已存在

**导入弹窗字段**：
- 上传文件（必填）
- 级别（必填，默认 moderate）
- 分组名（选填，填了则所有导入的词都归属该分组）

### 4.3 导出功能

**触发方式**：列表页顶部按钮「导出」

**格式**：TXT，格式与导入一致（空格分隔，行为与导入对应）

```
违禁词A 违禁词B
违禁词C
```

## 9. 认证与权限

- Filament 使用默认的 `users` 表认证（`AdminPanelProvider` 已配置 `->login()`）
- 中间件确保只有 `role = 'admin'` 的用户可以登录 Filament（`Authenticate` 中间件内检查 `role` 字段）
- 非 admin 用户访问 `/admin` 重定向至登录页
