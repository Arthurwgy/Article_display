#### 3.1 敏感词 Model + 迁移

> 原本：SensitiveWordController 5 个 API 接口
> 改后：Model + 迁移 + Filament Resource（不再做 API 层）

- [ ] `SensitiveWord` Model（参考 `database/migrations/2026_06_25_000008_create_sensitive_words_table.php`）

  违禁词D 违禁词E 违禁词F

- 硬币系统（每日任务）
- 打赏 + 付费阅读

### 任务清单

#### 5.1 关注

- [ ] `follows` 表：
  - `id` ulid
  - `follower_id` ulid
  - `following_id` ulid
  - `is_special` boolean（特殊关注，发文章触发通知）
  - timestamps
  - 唯一约束 `(follower_id, following_id)`
- [ ] `FollowController`：`POST /api/users/{id}/follow`、`DELETE /api/users/{id}/follow`
- [ ] 关注成功 → 给被关注者发通知（普通关注只在"我的关注"里展示，特殊关注触发通知）

#### 5.2 评论

- [ ] `comments` 表（按 `数据库设计.md`）
- [ ] `comment_likes` 表
- [ ] `CommentController`：
  - `GET /api/articles/{id}/comments`：树形（一级 + 二级嵌套）
  - `POST /api/articles/{id}/comments`：发评论，先走机审
- [ ] 评论点赞：`POST /api/comments/{id}/like`（轻量，鉴权即可）

#### 5.3 金币 + 硬币

**`gold_transactions` 表**（充值货币流水）：
- `id` ulid
- `user_id` ulid
- `type` enum('recharge', 'reward_out', 'reward_in', 'purchase', 'refund')
- `amount` decimal
- `balance_after` decimal
- `related_id` ulid nullable（关联文章等）
- timestamps

**`coin_transactions` 表**（硬币流水）：
- 同上结构，type 不同：`daily_login`, `article_passed`, `comment_liked`, `coin_in`, `coin_out`

**每日硬币**：
- [ ] `ArticleController::show` 或登录后触发
- [ ] 检查 `last_coin_at` 是否今天，是则跳过；否则 `coin_balance++` + 写流水 + 更新 `last_coin_at`

**每日投币上限 5 枚**：在 `POST /api/articles/{id}/coin` 接口里检查今日已投出数（`coin_transactions` 聚合）。

#### 5.4 打赏 + 付费阅读

- [ ] `article_purchases` 表：
  - `id` ulid
  - `user_id` ulid
  - `article_id` ulid
  - `gold_amount` int
  - `purchased_at` timestamp
  - 唯一 `(user_id, article_id)`
- [ ] `POST /api/articles/{id}/reward`：扣打赏者金币 + 给作者 70% + 平台 30%
- [ ] `POST /api/articles/{id}/purchase`：扣读者金币 + 给作者 70% + 平台 30% + 写 `article_purchases`
- [ ] **充值接口**：`POST /api/wallet/recharge`，MVP 直接加金币，不接真实支付
- [ ] **下架退款**：放后续迭代，M5 不做

#### 5.5 触发通知

- 评论文章 → 给作者发通知
- 被打赏 → 给作者发通知
- 被付费阅读 → 给作者发通知
- 被关注（特殊关注） → 给被关注者发通知

### 错误处理约定

- 业务错误：返回 `{ "message": "...", "errors": {...} }`，HTTP 400/403/422
- 找不到资源：404 + `{ "message": "Resource not found" }`
- 未认证：401
- 服务器错误：500 + 不暴露细节（生产环境）

### 验证规则（FormRequest）

每个写操作都建一个 FormRequest：
- `RegisterRequest`
- `LoginRequest`
- `StoreArticleRequest`
- `UpdateArticleRequest`
- `ReviewArticleRequest`
- 等等

### 测试

每个阶段至少跑通 curl 验证脚本。M3 之后可以加 PHPUnit Feature Test，但 MVP 不强制。

### 性能注意

- 文章列表：必须 `with(['user', 'category', 'tags'])` 预加载，避免 N+1
- 详情接口：渲染 Markdown 是 CPU 密集操作，可以加缓存（`Cache::remember`）
