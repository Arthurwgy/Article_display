# 前端执行方案（UniApp + Vue 3）

> 文档版本 1.0，2026-06-25
> 上游：`Docs/总需求文档.md` + `Docs/接口文档.md`
> 当前工程：`apps/`（Hello uni-app 模板，已清理过 uniCloud 模块）
> 技术栈：UniApp + Vue 3 + Vite + Pinia + uView UI（按需）

---

## 阶段总览

| 阶段 | 名称 | 接口对应 | 关键页面 | 预计产出 |
|---|---|---|---|---|
| M0 | 项目骨架 + 认证 | P0-1 | login, register, mine | 登录注册闭环 |
| M1 | 文章 CRUD | P0-2 | 列表, 详情, 编辑器, 我的文章 | 文章业务闭环 |
| M2 | 审核状态追踪 | P0-3 | 文章状态页, 审核日志 | 提交/申诉/重提 |
| M3 | 审核工作台 + 敏感词 + 分类标签 | P1-6/P1-7 | 管理员工作台, 敏感词管理 | 管理员视角闭环 |
| M4 | 通知 | P2 | 通知中心, 红点 | 通知闭环 |
| M5 | 关注 + 评论 + 付费 + 硬币 | P3 | 关注按钮, 评论组件, 打赏弹窗 | 完整 MVP |

## M0 — 项目骨架 + 认证（P0-1）

- 后端 M0 已跑通（4 个认证接口可用）
- HBuilderX 工程 `apps/` 已清理（uniCloud 模块已删除）

### 目标

- API 请求封装 + 用户状态管理 + 登录注册页面
- 登录成功后能跳转到首页
- 退出登录后跳回登录页

### 任务清单

#### 0.1 目录与配置

- [ ] `apps/config/index.js`：放 `BASE_URL`（开发：`http://127.0.0.1:8000/api`）
- [ ] `apps/utils/request.js`：基于 `uni.request` 封装
  - 自动加 `Authorization` header
  - 统一处理 401（清 token + 跳登录）
  - 统一错误提示（uni.showToast）
  - 返回 Promise
- [ ] `apps/utils/auth.js`：token 存取（`uni.getStorageSync`）
- [ ] `apps/utils/index.js`：导出

#### 0.2 状态管理（Pinia）

- [ ] `apps/store/modules/user.js`：state: `{ token, userInfo }`，actions: `login, register, logout, fetchMe`
- [ ] `apps/store/index.js`：createPinia 并导出

**在 `main.js` 已经引用了 Pinia**，确认能用。

#### 0.3 API 模块

- [ ] `apps/api/auth.js`：
  ```js

    register: (data) => request.post('/auth/register', data),
    login: (data) => request.post('/auth/login', data),
    logout: () => request.post('/auth/logout'),
  ```

#### 0.6 首页（M0 占位）

- `apps/pages/index/index.vue`：
  - 显示"欢迎来到 xxx"
  - 一个 tabBar 跳到 mine

#### 0.7 配置 tabBar

## M1 — 文章 CRUD（P0-2）

### 前置

- 后端 M1 已跑通
- 后端 M3 中 `GET /api/categories` 和 `GET /api/tags` 也可用（M1 里先做只读）

### 目标

- 文章列表（首页）
- 文章详情（Markdown 渲染）
- 文章编辑器（写文章）
- 我的文章列表

### 任务清单

#### 1.1 API 模块

- `apps/api/article.js`：
  ```js
  export const articleApi = {
    list: (params) => request.get('/articles', { params }),
    detail: (id) => request.get(`/articles/${id}`),
    create: (data) => request.post('/articles', data),
    update: (id, data) => request.put(`/articles/${id}`, data),
    remove: (id) => request.delete(`/articles/${id}`),
    submit: (id) => request.post(`/articles/${id}/submit`),
  }
  ```

- `apps/api/category.js`：list（分类树）
- `apps/api/tag.js`：list（标签列表 + 搜索）

#### 1.2 Markdown 渲染

- [ ] 安装：`markdown-it`、`dompurify`（用于 H5）
- [ ] `apps/utils/markdown.js`：封装 render 函数
- [ ] H5 端：直接用 v-html + DOMPurify
- [ ] 小程序/App 端：用 `mp-html` 或 `towxml`（后续阶段处理，M1 先只支持 H5）

#### 1.3 文章列表页

- `apps/pages/article/list.vue`（= 首页）：
  - 调用 `articleApi.list({ page: 1, per_page: 20 })`
  - 列表项：封面图、标题、作者头像+昵称、分类、标签、view_count
  - 置顶/精选标识
  - 付费标识（金币图标）
  - 下拉刷新 + 上拉加载
- 顶部搜索栏：`q` 参数，搜索标题+作者名
- 顶部筛选：分类 tab + 排序（最新/最热）

#### 1.4 文章详情页

- `apps/pages/article/detail.vue`：
  - 调用 `articleApi.detail(id)`
  - 头部：封面、标题、作者、发布时间、view_count、review_count
  - 正文：v-html 渲染 Markdown
  - 付费文章未付费：显示遮罩 + "支付 X 金币解锁"按钮
  - 底部操作栏：编辑（仅作者）、提交审核、删除（仅 draft）
  - 状态显示：草稿/初审中/已发布等（badge）

#### 1.5 文章编辑器

- `apps/pages/article/edit.vue`：
  - 标题输入
  - Markdown 编辑器（多行 textarea + 预览 tabs）
  - 分类选择（picker 调 `categoryApi.list`）
  - 标签输入（输入即创建，传给后端）
  - 封面图 URL 输入（M1 不做上传）
  - 金币定价输入
  - 底部按钮：保存草稿、提交审核

**Markdown 编辑器**：MVP 用 `<textarea>` + 预览 tab 就够了，富文本编辑器放后续。

#### 1.6 我的文章列表

- `apps/pages/mine/articles.vue`：
  - 调用 `articleApi.list({ author_id: me.id })`
  - 状态分组：草稿 / 审核中 / 已发布 / 已驳回
  - 点击进详情

#### 1.7 更新 tabBar 与页面

- 首页（列表）+ 我的（mine）+ 写文章（不放 tabBar，从 mine 入口进）
- `pages.json` 调整 tabBar

#### 1.8 状态徽章组件

- `apps/components/article-status-badge/article-status-badge.vue`：
  - 接收 `status` prop
  - 不同状态不同颜色：草稿-灰、初审中-黄、已发布-绿、初审驳回-红、要求修改-橙、申诉中-蓝、二审通过-绿、二审驳回-红

### 验收清单

- [ ] 首页能看到已发布文章
- [ ] 搜索能筛出结果
- [ ] 点文章进详情，Markdown 渲染正常

## M2 — 审核状态追踪（P0-3）

### 任务清单

- `apps/api/review.js`：
  ```js

    logs: (articleId) => request.get(`/articles/${articleId}/review-logs`),
  ```

  ```js
  export const adminApi = {

      update: (id, data) => request.put(`/admin/sensitive-words/${id}`, data),
      remove: (id) => request.delete(`/admin/sensitive-words/${id}`),
      import: (filePath) => request.upload('/admin/sensitive-words/import', { file: filePath }),
      exportUrl: () => `${BASE_URL}/admin/sensitive-words/export`,

  }
  ```

- `apps/api/category.js`：增删改（管理员用）

## 通用规范

### 命名约定

- 页面文件：kebab-case（`login.vue`, `article-detail.vue`）
- 组件文件：kebab-case，目录也用 kebab-case（`components/article-card/`）
- 工具/服务：camelCase（`request.js`, `markdown.js`）
- API 模块：camelCase + `Api` 后缀（`articleApi`, `authApi`）
- Store 模块：camelCase（`userStore`, `notificationStore`）

### 文件组织

### 状态管理原则

- **Pinia** 为主
- **本地状态**（页面内）：用 `ref` / `reactive`
- **跨页面状态**（用户信息、未读数）：Pinia
- **服务端数据**：不存 Pinia，组件内 onMounted/onShow 拉取，简单缓存可用 `useStorage`（pinia-plugin-persistedstate）

### API 错误处理

`utils/request.js` 统一处理：
- `code === 401`：清 token + 跳登录
- `code === 403`：toast 提示
- `code === 422`：显示 validation 错误
- `code === 5xx`：toast "服务器错误"
- 网络错误：toast "网络异常"

### 表单校验

- 用 uni-app 自带的 form 组件 + 简单正则
- 复杂校验用 async-validator

### Markdown 编辑器

- MVP：`<textarea>` + 预览 tab
- 进阶：`<md-editor>` 或类似组件

### UI 库

- 推荐 **uView Plus**（Vue 3 兼容）
- 或 **uni-ui**（DCloud 官方）
- MVP 不强装，原生 CSS 写也行

### 适配

- MVP 只做 H5，按设计稿写
- 小程序/App 适配放后续阶段（接口、状态管理都已平台无关，只剩样式和组件）

## 阶段性验收

每个 M 完成后自查：

| 阶段 | 自检命令/动作 |
|---|---|
| M0 | 注册 → 登录 → 看 mine 页面 |
| M1 | 首页看列表 → 详情看 Markdown → 写一篇保存草稿 |
| M2 | 提交 → 状态显示正确 → 申诉/重提 |
| M3 | 管理员视角：审一篇文章 → 用户看到状态变 |
| M4 | 收到通知 → 红点 → 已读 |
| M5 | 关注 + 评论 + 打赏 + 付费 + 硬币 全跑通 |

任何一步不通都先回滚，不要带病前进。
