### 验证方式

#### 自动化 tinker 检查

```php
use App\Enums\ArticleStatus;
use App\Services\ArticleStateMachine;
use App\Models\SensitiveWord;
use App\Models\Article;
use App\Models\ArticleReviewLog;

$sm = app(ArticleStateMachine::class);

// 1. UNLISTED 状态存在
echo ArticleStatus::UNLISTED->label();  // 已下架

// 2. 下架生命周期
echo $sm->canTransition(ArticleStatus::PUBLISHED, ArticleStatus::UNLISTED);  // true
echo $sm->canTransition(ArticleStatus::UNLISTED, ArticleStatus::PENDING);    // true

// 3. 修改要求动作可用（回归）
echo $sm->canTransition(ArticleStatus::PENDING, ArticleStatus::MODIFY_REQUIRED);  // true

// 4. SensitiveWord 可读写
$s = SensitiveWord::create(['word' => 'test_'.uniqid(), 'level' => 'moderate']);
echo $s->id;
$s->delete();

// 5. Article reviewLogs 关联可用
$a = Article::first();
echo $a->reviewLogs()->count();
```

#### 端到端浏览器验证

1. **登录**：访问 `http://127.0.0.1:8000/admin`，用 `admin@example.com / password` 登录
2. **拒绝访问**：用 `test@example.com`（role=reader）登录后访问 `/admin/articles` → 应得 403
3. **敏感词导入**：在 `/admin/sensitive-words` 点击「批量导入」，上传 `docs/fixtures/sensitive-words-sample.txt`（开发准备一个 10 行示例文件），确认导入成功提示与数据库条数一致
4. **审核流程**：
   - 用 `author@example.com` 登录会员端，写一篇文章 → 提交
   - 切回 admin 账号，访问 `/admin/articles`，筛选状态=pending
   - 点击文章进编辑页，点击「要求修改」→ 填 reason → 提交
   - 确认文章状态变为 `modify_required`，且「审核记录区域」出现一条新日志
5. **下架**：把文章从 `pending` 改为 `published`，编辑页出现「下架」按钮 → 点击 → 确认状态变为 `unlisted`，列表 badge 变灰
6. **敏感词命中回显**：在 `/admin/review-logs` 找到刚生成的日志，`sensitive_word_hit` 列正确展示命中的词数组

### 不在 M3 范围（避免越界）

- 树形分类的拖拽排序（默认缩进展示已够用）
- 批量操作（批量通过/驳回）
- 管理员内部备注（`admin_note` 字段，复用 `reason` 即可）
- 用户的「封号/解封」动作（数据库设计文档未列该字段）
- 文章编辑器（管理员查看用 MarkdownEntry 只读预览，写作用户端编辑器）
- 实时通知（管理员操作后给作者发邮件/站内信是 M4 通知阶段）
