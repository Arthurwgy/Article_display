## 通用约定

1. **接口契约**：所有接口以 `Docs/接口文档.md` 为准。任何变更先改文档再改代码。
2. **数据库**：以 `Docs/数据库设计.md` 为准。迁移文件命名 `YYYY_MM_DD_HHMMSS_xxx.php`。
3. **错误格式**：所有 API 错误统一返回 `{ "message": "...", "errors": {...} }`。
4. **ID 类型**：字符串 ULID（26 位），用户/文章/通知/分类等业务表统一。
5. **认证**：JWT 无刷新 Token，`Authorization: Bearer <token>`，过期 1 小时。
6. **开发环境**：
   - 后端：`php artisan serve` 默认 `http://127.0.0.1:8000`
   - 前端 H5：`npm run dev:h5` 默认 `http://localhost:5173` / `8080` / `9000`
   - CORS：Laravel 11+ 已内置，前端开发时如遇跨域在 `config/cors.php` 配置 `allowed_origins`（白名单已固定含以上三个端口，见 `2后端/后端-决策记录.md` D5）

## 分阶段里程碑

| 阶段 | 后端产出 | 前端产出 | 验证方式 |
|---|---|---|---|
M0 | Laravel 项目初始化 + JWT 安装 + 用户迁移 + 认证接口 | UniApp 模板清理 + 基础结构 | `POST /api/auth/register` 拿 token，`GET /api/auth/me` 拿到用户 |
| **M1** | 文章表 + CRUD 接口 + Markdown 渲染 | 文章列表/详情/编辑器 | 写一篇文章 → 后端入库 → 前端拉列表能看到 |
| **M2** | 审核状态机 + 审核日志 + 机审 | 用户端审核状态页 + 提交/申诉/重提 | 提交 → 初审通过 → 已发布，全链路通 |
| **M3** | 敏感词 CRUD + 导入导出 + 审核工作台 + 分类标签（Filament Resource） | 不做前端管理页面 | 管理功能在 Filament 后台完成（/admin） |
| **M4** | 通知表 + 触发逻辑 + 邮件 | 通知中心页面 + 未读红点 | 发文章审核通过 → 收到站内信 |
| **M5** | 关注 + 评论 + 付费（金币） + 硬币 | 关注按钮 + 评论组件 + 打赏/付费弹窗 | 全功能 MVP 跑通 |
> 原本：M3 后端对应前端「审核工作台 H5 页面（管理员）」
> 改后：M3 管理功能由 Filament Resource 提供，前端 M3 改为通知（M4→M3），M4 改为关注+评论+付费（M5→M4）

每个 M 是**独立可交付**的，不依赖后续阶段的产出。

## AI 协作约定

- **每个阶段开新对话**：上下文不互相污染。
- **每个对话开始时**：贴"我正在执行 `Docs/执行计划/xxx.md` 的第 X 阶段"，让 AI 知道边界。
- **每个阶段结束前**：跑通"验证方式"那一栏，确认通过再进入下一阶段。
- **遇到超出文档的问题**：先回来更新 `Docs/`，再改代码。
