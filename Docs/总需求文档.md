- 内容发布平台，用户写文章提交审核，审核通过后发布
- 内容社区：关注、评论、打赏、付费阅读

## 3. 用户体系

- **统一用户模型**：注册后默认既是读者也是作者，无需角色切换
- 注册方式：邮箱 + 密码
- 认证方式：JWT（无刷新 token）
- 用户表：`users` 表，`role` 字段区分（`reader` / `author` / `admin`）
- 管理员：Filament 默认 admin 账号兼任审核，不另建审核员角色
- 无第三方登录，无手机号登录

---

## 4. 文章系统

### 4.1 基础

- 编辑器：**Markdown**（文章正文）+ **富文本**（评论用）
- 支持元素：多图上传、代码高亮、附件下载（导出 `.md`）、音乐嵌入
- 封面图：优先用户上传，无则自动提取文章第一张图
- 分类：**多级分类**（后台可配置深度）
- 标签：**用户自由创建**（写文章时随手新建，类似微博），不做预置

### 4.2 文章状态

`草稿 → 初审中 → 初审通过 → 已发布`
`         → 初审驳回 → 修改后重新提交`
`         → 要求修改 → 修改后重新提交`
`         → 申诉二审 → 二审通过 → 已发布`
`                → 二审驳回 → 最终驳回`

- 已发布文章不能删除，只能申请下架
- 编辑已发布文章 → 需重新走审核流程
- 修改次数：**无限次**，每次记录审核次数
- 版本历史：每次「要求修改」时自动保存快照，以审核次数为版本号

### 4.3 列表与搜索

- 列表分页：**传统页码**（1, 2, 3 ...）
- 支持置顶 / 精选推荐
- 搜索：文章标题 + 作者名
- 付费文章未付款前：**只能看到标题 + 封面图**

## 5. 审核流程（二级）

### 5.1 流程节点

1. 用户提交 → 机审（敏感词）→ 通过则进入初审队列
2. **初审**：通过 / 驳回（附理由）/ 要求修改
3. 用户申诉 → **二审**（由初审员自己做）
4. 二审通过 → 发布；二审驳回 → 最终驳回

### 5.2 审核动作

| 动作 | 说明 |
|---|---|
| 通过 | 进入已发布 |
| 驳回 | 附理由，用户可申诉 |
| 要求修改 | 附理由，保存版本快照，用户修改后重新提交 |
| 申诉 | 进入二审队列 |
| 二审通过 | 进入已发布 |
| 二审驳回 | 最终结果，不可再申诉 |

### 5.3 敏感词机审

- 后台管理敏感词：增删改、导入/导出（Excel / TXT）
- **三级分组**：
  - 轻度：替换为 `***`，继续流转
  - 中度：自动驳回
  - 重度：自动驳回
- 机审匹配到 → **直接自动驳回**（轻度替换为 `***` 后再驳回提示）

## 6. 关注系统

- **单向关注**（微博/Twitter 模式，无需双向确认）
- 取关：**随时可取**，无冷却期
- 普通关注：仅出现在「我的关注」列表，**无站内信通知**
- **特殊关注**：用户主动勾选 → 作者发新文章 → 触发站内信通知（一篇一发）
- 作者主页展示：头像 + 昵称 + 简介 + 文章列表 + 关注/粉丝数 + 打赏收入统计

---

## 7. 付费系统（金币 + 硬币双货币）

### 7.1 金币（充值货币）

- 用途：打赏、付费阅读、付费专栏解锁
- 特点：用户用现金充值，有钱无限用
- 付费粒度：单篇文章 / 专栏（多篇打包，统一定价）/ 打赏（任意金额）
- 付费方式：**纯模拟**，界面有充值 UI，不接真实支付
- **分利规则**：作者得 70%，平台留 30%（平台留 30% 为运营费用，不返还用户）
- 下架退款：平台全额退还读者（包括作者的 70% 和平台的 30%）

### 7.2 硬币（平台积分 / 热度）

- 类似 B 站硬币，对应文章「热度投票」
- 获取渠道：
  - 每日首次打开 App：+1 枚
  - 投稿通过审核：+5 枚
  - 评论被点赞（每 2 个赞）：+1 枚
  - 作为作者收到投币：+少量（平台发放，非打赏者支出）
- 每日投币上限：**5 枚**（B 站标准）
- 投出即消耗，**不可撤回**

## 8. 评论系统

- 结构：**一级评论 + 二级嵌套**（一层）
- 审核：**先审后发**（共用文章的敏感词库）→ 展示 → 可举报 → 人工处理
- 点赞：支持
- 管理：作者可删除评论 / 拉黑用户
- 时间：**文章发布后任意时间**可评论，无关闭机制

---

## 9. 消息通知

### 9.1 渠道

- **站内通知**：数据库记录，红点未读数，支持标记已读
- **邮件通知**：Laravel `mail` driver，开发阶段用 `log` 驱动（打印到日志）

### 9.2 触发场景

- 文章审核结果（通过 / 驳回 / 要求修改 / 二审结果）
- 文章被评论
- 被关注（包括特殊关注的站内信）
- 被打赏 / 付费阅读
- 系统公告

---

## 10. 音乐嵌入

- 来源：**外链音频**（粘贴第三方链接，如网易云单曲）
- 不上传音频文件到服务器（避免存储和带宽成本）
- 实现方式：编辑页「插入音乐」按钮 → 弹窗填「封面图 URL」+「音乐链接」→ 存储两个字段 → 渲染用 `<audio>` 播放
- 小程序注意：部分平台外链音频有域名限制（后续扩展时处理）

---

## 11. 安全与 XSS

- 文章内容：**不防爬**，随便抓
- Markdown 存储：存原始 Markdown，前端用 `markdown-it` + HTML Purifier 渲染
- 文件上传：无格式限制，无大小限制
