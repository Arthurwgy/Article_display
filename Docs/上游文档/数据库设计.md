## 1. 概览

| 维度 | 说明 |
|---|---|
| 数据库 | MySQL 8.0+ |
| ORM | Laravel Eloquent |
| 迁移目录 | `database/migrations/` |
| 设计原则 | 尽量范式化、避免冗余；状态/枚举用 `enum` 或 `string` 而非布尔；所有 `id` 字段用 `ulid` 或 `bigIncrements` 视规模而定 |

## 3. 表结构详细定义

### 3.1 users（用户表）

| 字段 | 类型 | 说明 |
|---|---|---|
| id | bigIncrements | PK |
| name | string(50) | 昵称，唯一 |
| email | string(255) unique | 登录账号 |
| password | string(255) | bcrypt |
| role | enum('reader','author','admin') | 默认 reader |
| avatar | string(500) nullable | 头像 URL |
| bio | text nullable | 个人简介 |
| last_login_at | timestamp nullable | 上次登录时间 |
| last_coin_at | date nullable | 上次领每日硬币日期 |
| email_verified_at | timestamp nullable | 邮箱验证时间 |
| remember_token | string(100) nullable | |
| created_at/updated_at | timestamp | |
| deleted_at | softDeletes | |

> 说明：
> - `role` 仅区分权限层级，不做业务隔离（注册即同时是读者也是作者）。
> - `gold_balance`、`coin_balance` **不存**在 users 表，改由 `user_wallets` 表统一管理（避免多表更新）。
> - 管理员使用 Filament 自带的 `filament_users` 表（Filament 5 内置）或共用的 `users` 表均可，建议共用以简化查询。

---

### 3.2 articles（文章表）

| 字段 | 类型 | 说明 |
|---|---|---|
| id | bigIncrements | PK |
| user_id | bigInteger FK → users.id | 作者 |
| category_id | bigInteger FK → article_categories.id nullable | 分类 |
| status | enum('draft','pending','pending_approved','published','rejected','revision_required','second_review','final_rejected','unlisted') | 文章状态 |
> 注意：这是设计稿 enum，实际迁移文件和接口文档使用简化命名 `first_pass` / `first_reject` / `second_pass` / `second_reject` / `appealing`，以迁移文件和接口文档为准
| title | string(255) | 标题 |
| slug | string(300) unique | URL slug（标题英文化） |
| content | longText | 原始 Markdown 内容 |
| cover_image | string(500) nullable | 封面图 URL（优先用户上传，无则自动提取正文第一张图） |
| is_top | boolean default false | 是否置顶 |
| is_featured | boolean default false | 是否精选推荐 |
| price_gold | decimal(10,2) default 0 | 付费阅读金币价，0=免费 |
| view_count | bigInteger default 0 | 浏览计数 |
| review_count | integer default 0 | 审核次数（每次提交 +1） |
| last_review_at | timestamp nullable | 最后提交审核时间 |
| published_at | timestamp nullable | 发布时间（初审通过/二审通过时写入） |
| deleted_at | softDeletes | 软删除（已发布文章不能真删，只能 unlisted） |
| created_at/updated_at | timestamp | |

> 说明：
> - `status` 用 enum 约束 9 种状态，避免状态值乱飞。
> - 已发布文章"不能删除"通过 `deleted_at` 软删除约束实现：发布后不允许调用 `forceDelete()`，只允许 `unlisted`（改 `status`）。

### 3.6 article_review_logs（审核记录表）

| 字段 | 类型 | 说明 |
|---|---|---|
| id | bigIncrements | PK |
| article_id | bigInteger FK → articles.id | 文章 ID |
| reviewer_id | bigInteger FK → users.id nullable | 审核人 ID（null=系统机审） |
| action | enum('submit','auto_reject','approve','reject','revision_required','appeal','second_approve','second_reject','publish','unlist') | 审核动作 |
> 注意：这是设计稿 enum，实际迁移文件使用简化命名 `first_pass` / `first_reject` / `second_pass` / `second_reject` / `appealing`，以迁移文件和接口文档为准
| reason | text nullable | 理由（驳回/要求修改时必填） |
| review_round | integer default 1 | 审核轮次（第 N 次审核） |
| sensitive_word_hit | json nullable | 机审命中的敏感词列表（仅 submit/auto_reject 时记录） |
| created_at | timestamp | |

> 说明：
> - `submit`（提交审核）、`auto_reject`（机审驳回）、`approve`（初审通过）、`reject`（初审驳回）、`revision_required`（要求修改）、`appeal`（申诉）、`second_approve`（二审通过）、`second_reject`（二审驳回）、`publish`（管理员直接发布）、`unlist`（申请下架）。
> - `review_round` 与 `articles.review_count` 保持一致。

### 3.7 article_snapshots（版本快照表）

| 字段 | 类型 | 说明 |
|---|---|---|
| id | bigIncrements | PK |
| article_id | bigInteger FK → articles.id | 文章 ID |
| review_round | integer | 对应审核轮次 |
| title_snapshot | string(255) | 快照时的标题 |
| content_snapshot | longText | 快照时的正文 Markdown |
| cover_image_snapshot | string(500) nullable | 快照时的封面图 |
| created_at | timestamp | |

> 说明：仅在审核员执行"要求修改"(`revision_required`)时自动创建快照。每个审核轮次最多一条快照。

### 3.8 sensitive_words（敏感词表）

| 字段 | 类型 | 说明 |
|---|---|---|
| id | bigIncrements | PK |
| word | string(100) | 敏感词 |
| level | enum('light','moderate','severe') | 级别：轻度/中度/重度 |
| group_name | string(50) nullable | 分组名（便于管理，如"政治类"、"广告类"） |
| created_at/updated_at | timestamp | |

> 说明：
> - 轻度(light)：替换为 `***`，文章继续流转到初审队列。
> - 中度(moderate)/重度(severe)：自动驳回，文章进入"初审驳回"状态。
> - 导入导出格式：TXT，以空白符（空格/制表符/换行）拆分 token，每 token 为一个独立敏感词（见 `Docs/管理端.md`）
> - 不再使用 `maatwebsite/excel`，无需 Excel 导入
>
> > 原本：支持 Excel/TXT 批量导入/导出（`maatwebsite/excel`）
> > 改后：TXT 空格分隔，不依赖 Excel 插件

### 3.9 follows（关注表）

| 字段 | 类型 | 说明 |
|---|---|---|
| id | bigIncrements | PK |
| follower_id | bigInteger FK → users.id | 关注者 |
| following_id | bigInteger FK → users.id | 被关注者 |
| is_special | boolean default false | 是否特殊关注（勾选后有新文章发站内通知） |
| created_at | timestamp | |

> 唯一约束：(follower_id, following_id)。
> 说明：单向关注，A 关注 B 不需要 B 同意。`is_special` 独立控制，避免每次都要改关系。

### 3.12 gold_transactions（金币流水表）

| 字段 | 类型 | 说明 |
|---|---|---|
| id | bigIncrements | PK |
| from_user_id | bigInteger FK → users.id nullable | 付款方（recharge 时为 null） |
| to_user_id | bigInteger FK → users.id nullable | 收款方（refund 时为 null） |
| type | enum('recharge','reward','read','refund','payout') | 交易类型 |
| amount | decimal(10,2) | 金额（正数） |
| article_id | bigInteger FK → articles.id nullable | 关联文章（打赏/付费阅读时填写） |
| order_no | string(64) unique | 订单号（用于幂等） |
| split_data | json nullable | 分账详情（`{"author": 70, "platform": 30}`） |
| created_at | timestamp | |

> 说明：
> - `reward`：打赏，作者得 70%，平台留 30%。
> - `read`：付费阅读，读者付金币给作者，同 70/30 分账。
> - `refund`：退款（下架），全额从平台账户退回读者。
> - `payout`：作者提现（后续迭代）。

### 3.13 coin_transactions（硬币流水表）

| 字段 | 类型 | 说明 |
|---|---|---|
| id | bigIncrements | PK |
| user_id | bigInteger FK → users.id | 用户 ID |
| type | enum('login_daily','article_published','comment_liked','article_received','platform_grant') | 硬币获取类型 |
| amount | integer | 数量（正数） |
| article_id | bigInteger FK → articles.id nullable | 关联文章 |
| created_at | timestamp | |

> 说明：
> - `login_daily`：每日首次打开 App，+1（需检查 `users.last_coin_at`）。
> - `article_published`：投稿通过审核，+5。
> - `comment_liked`：评论被点赞，每 2 个赞 +1（累积计算）。
> - `article_received`：作为作者收到硬币，平台发放少量。
> - 硬币**投出即消耗**：投币时扣减 `user_wallets.coin_balance`，记录 `amount` 为负数或不记录（改为单独 `coin_votes` 表）。

## 4. 文章状态机

> 注意：这是设计稿状态机，使用旧命名（`pending_approved`/`rejected`/`revision_required`/`second_review`/`final_rejected`）。实际迁移文件和接口文档已简化为 `first_pass` / `first_reject` / `second_pass` / `second_reject` / `appealing`，以迁移文件和接口文档为准。

### 4.1 状态枚举值

| 值 | 含义 | 用户可见 | 管理员可见 |
|---|---|---|---|
| draft | 草稿 | ✅ | ✅ |
| pending | 初审中 | ✅ | ✅ |
| pending_approved | 初审通过，待发布 | ✅ | ✅ |
| published | 已发布 | ✅ | ✅ |
| rejected | 初审驳回（可申诉）| ✅ | ✅ |
| revision_required | 要求修改（保存快照）| ✅ | ✅ |
| second_review | 申诉二审中 | ✅ | ✅ |
| final_rejected | 二审驳回（不可再申诉）| ✅ | ✅ |
| unlisted | 申请下架 | ✅ | ✅ |

### 4.2 状态流转图

```
用户提交
  │
  ▼
┌──────────────┐  机审命中(中/重度) ┌────────────────┐
│    draft     │──────────────────→│  rejected(自动) │
└──────┬───────┘                   └────────────────┘
       │ 机审通过
       ▼
┌──────────────┐   初审通过
│   pending    │───────────────────────────────────→ (机审命中轻度→替换***，继续流转)
└──────┬───────┘
       │ 初审员操作
       ▼
  ┌────┴────┐
  │approve  │  ← 初审通过
  └────┬────┘
       │                          ┌────────────────────┐
       ▼                          │  pending_approved  │  ← 待发布(可选)
  ┌─────────┐                     └────────────────────┘
  │published│                              │
  └────┬────┘                              │ 管理员发布 or 自动
       │ 编辑已发布文章                      │ 发布
       │ (increment review_count)            ▼
       ▼                               ┌─────────┐
  ┌────────────┐                       │published│
  │  pending   │ (重新进入审核)          └─────────┘
  └────────────┘
       │
       │ 初审员操作
  ┌────┴──────────────────┐
  │reject │revision_req  │appeal
  ▼       ▼              ▼
┌────────┐ ┌────────────┐ ┌────────────────┐
│rejected│ │revision_req│ │  second_review │
└───┬────┘ └─────┬──────┘ └───────┬────────┘
    │ 用户申诉   │保存快照         │ 二审员操作
    ▼           ▼                 ▼
┌────────────────┐         ┌─────┴──────┐
│ second_review  │         │second_approve│second_reject
└───────┬────────┘         └─────┬──────┘
        │                        │
        │ 二审员操作             ▼
  ┌─────┴──────┐          ┌──────────┐
  │second_    │          │final_    │
  │approve    │          │rejected  │
  └─────┬──────┘          └──────────┘
        │
        ▼
  ┌─────────┐
  │published│
  └─────────┘
```

### 4.3 流转规则总结

| 当前状态 | 可执行动作 | 目标状态 |
|---|---|---|
| draft | submit | pending |
| pending | approve | pending_approved / published |
| pending | reject | rejected |
| pending | revision_required | revision_required |
| pending (机审中/重度命中) | auto_reject | rejected |

| second_review | second_approve | published |
| second_review | second_reject | final_rejected |
| published | edit | pending (review_count+1) |
| published | unlist_request | unlisted |

## 7. 实现注意事项

1. **状态机不写在 Model 内**，而是 `app/Services/ArticleStateMachine.php`，用 `match` 语句实现，职责单一，便于测试。
2. **敏感词机审**在 `ArticleSubmitListener`（`created` 事件）内触发，命中后自动写入 `article_review_logs`。
3. **硬币每日领取**在 `DailyCoinGrantListener`（用户认证后）触发，检查 `users.last_coin_at`。
4. **文章版本快照**在 `ArticleRevisionRequiredListener`（审核动作 `revision_required`）内触发。
5. **70/30 分账**在 `GoldTransactionCreated` 事件监听器内原子执行：先写 `gold_transactions`（两条记录），再原子更新 `user_wallets`（DB Transaction）。
6. **付费阅读拦截**：在 `ArticlePolicy` 的 `view` 方法内检查 `price_gold > 0` 且用户不在 `article_paid_access` 列表中，返回 403。
